OrderedDictionary

A generic dictionary that preserves the order in which keys are inserted.

Entries are stored in a doubly linked list, so insertion, replacement, and removal are all O(1). Positional lookups (keyAtIndex/indexAtKey) are served from an index cache that is rebuilt lazily on the first lookup after a structural change, making repeated lookups O(1) until the next mutation. 

local dict = OrderedDictionary.new()
dict:set("first", 1)
dict:set("second", 2)

print(dict:indexAtKey("second")) --> 2
print(dict:keyAtIndex(1)) --> "first", 1

set​

</>
OrderedDictionary:set(
key: K,
value: V
) → ()

Sets the value for key.

  • If the key is new, it is appended to the end of the order.
  • If the key already exists, its value is replaced in place, keeping its position.
  • If value is nil, the key is removed from the dictionary.

keyAtIndex​

</>
OrderedDictionary:keyAtIndex(index: number) → (
K,
V
)

Returns the key and value at the given 1-based position in the order.

Positive indices count from the front (1 is the first entry); negative indices count from the back (-1 is the last entry). Throws if index is zero or its magnitude exceeds the length.

indexAtKey​

</>
OrderedDictionary:indexAtKey(key: K) → number?

Returns the 1-based position of key in the order, or nil if the key is not present.

iterate​

</>
OrderedDictionary:iterate(reversed: boolean?) → () → (
K?,
V?,
number?
)

Returns an iterator over the entries in insertion order, suitable for use in a generic for loop. Pass true to iterate from last to first instead.

Each step yields the key, the value, and the entry's 1-based position in the order (the same value OrderedDictionary:indexAtKey would return for that key, regardless of direction). 

for key, value, index in dict:iterate() do
	print(index, key, value)
end
